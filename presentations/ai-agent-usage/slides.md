# 为什么需要 Spec-Driven？

<v-clicks>

1. **需求与实现的鸿沟** - 传统文档难以精确表达意图
2. **AI 需要结构化输入** - 自然语言不够精确
3. **可验证性** - Spec 可以被测试和验证
4. **可维护性** - Spec 即文档，永远同步

</v-clicks>

# 实践案例

<v-clicks>

## 1. API 开发
- OpenAPI Spec → 自动生成服务端代码
- 包含路由、验证、文档

## 2. 数据处理流程
- 定义数据 Schema → Agent 生成 ETL 管道
- 自动处理边界情况

## 3. 测试自动化
- 行为规范 (BDD) → Agent 生成测试用例
- 覆盖率自动优化

</v-clicks>
