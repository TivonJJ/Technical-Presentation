# 演讲前检查清单

## 技术准备

- [ ] 运行 `npm install` 确保所有依赖已安装
- [ ] 运行 `npm run dev` 测试演示文稿能正常启动
- [ ] 检查所有图片资源是否正常显示
- [ ] 测试演讲者模式（按 `s` 键）
- [ ] 确认备注内容完整且准确
- [ ] 测试全屏模式（按 `f` 键）
- [ ] 准备备用 PDF 版本（`npm run export`）

## 内容检查

- [ ] 开场白流畅自然
- [ ] 每个技术概念都有清晰的定义
- [ ] 示例代码可读性良好
- [ ] 流程图和架构图清晰易懂
- [ ] 技术演进路线逻辑清晰
- [ ] 总结部分有力且完整

## 演讲技巧

- [ ] 熟悉快捷键操作
  - `Space` / `→` : 下一页/动画
  - `Shift+Space` / `←` : 上一页
  - `o` : 概览模式
  - `s` : 演讲者模式
  - `f` : 全屏模式
  - `d` : 暗黑模式切换
- [ ] 练习每页的过渡话术
- [ ] 准备可能的提问回答
- [ ] 控制每页的时间节奏

## 设备准备

- [ ] 笔记本电脑充满电
- [ ] 准备投影仪/显示器连接线
- [ ] 测试外接显示器效果
- [ ] 准备激光笔或演示笔（可选）
- [ ] 确保网络连接稳定（如需 Demo）

## 时间分配建议

| 章节 | 预计时间 |
|------|---------|
| 开场 + LLM 基础 | 3-5 分钟 |
| Context & Prompt Engineering | 5 分钟 |
| RAG & GraphRAG | 8-10 分钟 |
| ReAct / Function Calling / MCP | 10-12 分钟 |
| Agent & Multi-Agent | 10-12 分钟 |
| 总结 + Q&A | 5-8 分钟 |
| **总计** | **40-50 分钟** |

## 备用方案

- [ ] 准备离线版本（防止网络问题）
- [ ] 准备 PDF 版本（防止技术故障）
- [ ] 准备简化版讲稿（时间不足时）
- [ ] 准备扩展内容（时间充裕时）

## 演讲后

- [ ] 收集听众反馈
- [ ] 记录提问和回答
- [ ] 更新演讲稿中的不足之处
- [ ] 分享演讲资料给听众
